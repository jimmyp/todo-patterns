# Web UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the Razor Pages Web UI — a thin browser client that calls the Api over HTTP. The implementation must enforce authentication (cookie + OAuth social login), protect the Index page behind `[Authorize]`, provide a logout flow, surface operation polling in JavaScript, and pass all integration tests in `TodoList.IntegrationTests/Web/`.

**Architecture:** Web is a pure presentation layer. It holds no domain logic. All state lives in the Api. Mutations call the Api and redirect-after-post (PRG pattern). Background operation polling is optional progressive enhancement — the page is fully functional without JavaScript.

**Tech Stack:** .NET 10, ASP.NET Core Razor Pages 10.0, `AspNet.Security.OAuth.GitHub`, `Microsoft.AspNetCore.Authentication.Google`, WireMock.NET (integration tests only), `WebApplicationFactory<T>`

> **Everything that already exists:** `Program.cs`, `Services/TodoApiClient.cs`, `Pages/Index.cshtml`, `Pages/Index.cshtml.cs`, `Pages/Login.cshtml`, `Pages/Login.cshtml.cs`, `Pages/Shared/_Layout.cshtml`, `wwwroot/js/app.js`, `Fixtures/WebFixture.cs`, `Web/WebIndexTests.cs`. Read each before touching it — many tasks are amendments to existing files, not rewrites.

## What is missing vs. the spec

The current implementation is mostly correct but has these gaps:

1. **No `[Authorize]` on `Index.cshtml.cs`** — unauthenticated users can reach the todo list. Spec requires all Web pages behind auth; unauthenticated users must be redirected to `/Login`.
2. **No logout endpoint** — `Program.cs` sets `LogoutPath = "/Logout"` but there is no `Logout` Razor Page or handler that calls `SignOutAsync`.
3. **`WebFixture` uses `Testing` environment but no fake auth** — once `[Authorize]` is added, the integration tests will 302 to `/Login` and fail. The fixture must set up a pre-authenticated identity for the `Testing` environment (same `TestAuthHandler` pattern used in `TodoList.Api`).
4. **Integration tests don't assert on auth redirect** — a test for the unauthenticated redirect is missing.
5. **`_Layout.cshtml` has no nav/user info** — the layout should show the logged-in user's name and a logout link, consistent with "thin shell" Web design.

## Tasks

### Task 1: Create `TestAuthHandler` for the Web project

The `Testing` environment needs a fake authentication scheme so integration tests can reach the `[Authorize]`-protected Index page without real OAuth credentials. This mirrors the `TestAuthHandler` already in `TodoList.Api/Auth/`.

- [ ] **Step 1:** Read `TodoList.Api/Auth/TestAuthHandler.cs` to understand the existing pattern
- [ ] **Step 2:** Create `TodoList.Web/Auth/TestAuthHandler.cs`:

```csharp
using System.Security.Claims;
using System.Text.Encodings.Web;
using Microsoft.AspNetCore.Authentication;
using Microsoft.Extensions.Options;

namespace TodoList.Web.Auth;

/// <summary>
/// Fake authentication scheme used in integration tests (ASPNETCORE_ENVIRONMENT=Testing).
/// Always authenticates as a fixed test user — no real OAuth credentials required.
/// </summary>
public class TestAuthHandler(
    IOptionsMonitor<AuthenticationSchemeOptions> options,
    ILoggerFactory logger,
    UrlEncoder encoder)
    : AuthenticationHandler<AuthenticationSchemeOptions>(options, logger, encoder)
{
    public const string SchemeName = "Test";

    protected override Task<AuthenticateResult> HandleAuthenticateAsync()
    {
        var claims = new[]
        {
            new Claim(ClaimTypes.NameIdentifier, "test-user-id"),
            new Claim(ClaimTypes.Name,            "Test User"),
            new Claim(ClaimTypes.Email,           "test@example.com"),
        };
        var identity  = new ClaimsIdentity(claims, SchemeName);
        var principal = new ClaimsPrincipal(identity);
        var ticket    = new AuthenticationTicket(principal, SchemeName);
        return Task.FromResult(AuthenticateResult.Success(ticket));
    }
}
```

**Acceptance:** File exists, compiles, SchemeName is `"Test"`.

### Task 2: Wire `TestAuthHandler` in `Web/Program.cs`

The `Testing` environment must use `TestAuthHandler` as the default scheme so that the `WebFixture` gets an authenticated user without OAuth.

- [ ] **Step 1:** Read `TodoList.Web/Program.cs` to see current auth registration
- [ ] **Step 2:** Read `TodoList.Api/Program.cs` to see the exact if/else pattern used there
- [ ] **Step 3:** Update `Program.cs` — replace the current `builder.Services.AddAuthentication(...)` block with a conditional:

```csharp
// Authentication — Testing env uses a fake handler (integration tests, no OAuth required)
if (builder.Environment.IsEnvironment("Testing"))
{
    builder.Services
        .AddAuthentication(TodoList.Web.Auth.TestAuthHandler.SchemeName)
        .AddScheme<AuthenticationSchemeOptions, TodoList.Web.Auth.TestAuthHandler>(
            TodoList.Web.Auth.TestAuthHandler.SchemeName, _ => { });
}
else
{
    builder.Services
        .AddAuthentication(CookieAuthenticationDefaults.AuthenticationScheme)
        .AddCookie(opts =>
        {
            opts.LoginPath  = "/Login";
            opts.LogoutPath = "/Logout";
        })
        .AddGoogle(opts =>
        {
            opts.ClientId     = builder.Configuration["Auth:Google:ClientId"]     ?? "";
            opts.ClientSecret = builder.Configuration["Auth:Google:ClientSecret"] ?? "";
            opts.CallbackPath = "/signin-google";
        })
        .AddGitHub(opts =>
        {
            opts.ClientId     = builder.Configuration["Auth:GitHub:ClientId"]     ?? "";
            opts.ClientSecret = builder.Configuration["Auth:GitHub:ClientSecret"] ?? "";
            opts.CallbackPath = "/signin-github";
        });
}
```

### Task 4: Create `Logout` Razor Page

The cookie auth is configured with `LogoutPath = "/Logout"` but no page exists at that route. Users clicking "Logout" in the nav get a 404.

- [ ] **Step 1:** Create `TodoList.Web/Pages/Logout.cshtml.cs`:

```csharp
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Authentication.Cookies;
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;

namespace TodoList.Web.Pages;

public class LogoutModel : PageModel
{
    public async Task<IActionResult> OnGetAsync()
    {
        await HttpContext.SignOutAsync(CookieAuthenticationDefaults.AuthenticationScheme);
        return RedirectToPage("/Login");
    }

    public async Task<IActionResult> OnPostAsync()
    {
        await HttpContext.SignOutAsync(CookieAuthenticationDefaults.AuthenticationScheme);
        return RedirectToPage("/Login");
    }
}
```

- [ ] **Step 2:** Create `TodoList.Web/Pages/Logout.cshtml` — minimal, redirect-only page (the `OnGetAsync` redirects immediately so the view rarely renders, but Razor Pages requires a view file):

```cshtml

```

### Task 7: Add auth-redirect integration test

Add a test that verifies unauthenticated requests to `/` are redirected to `/Login`. This requires a second fixture or a client without the test auth handler.

- [ ] **Step 1:** Read `TodoList.IntegrationTests/Web/WebIndexTests.cs`
- [ ] **Step 2:** Add a new test class `WebAuthTests.cs` in `TodoList.IntegrationTests/Web/`:

```csharp
using Microsoft.AspNetCore.Hosting;
using Microsoft.AspNetCore.Mvc.Testing;
using TodoList.Web.Services;

namespace TodoList.IntegrationTests.Web;

/// <summary>
/// Verifies that unauthenticated requests are redirected to /Login.
/// Uses a separate factory with no TestAuthHandler (non-Testing environment).
/// </summary>
public class WebAuthTests : IDisposable
{
    private readonly WebApplicationFactory<TodoApiClient> _factory;
    private readonly HttpClient _client;

    public WebAuthTests()
    {
        _factory = new WebApplicationFactory<TodoApiClient>()
            .WithWebHostBuilder(host =>
            {
                // Do NOT set Testing environment — use Production-like auth
                // that requires real credentials (which we won't provide)
                host.UseEnvironment("Integration");
                host.ConfigureServices(services =>
                {
                    // Point the typed client at a non-existent address — won't be called
                    // because the auth redirect happens before any API call
                    services.AddHttpClient<TodoApiClient>(c =>
                        c.BaseAddress = new Uri("http://localhost:9999"));
                });
            });

        _client = _factory.CreateClient(new WebApplicationFactoryClientOptions
        {
            AllowAutoRedirect = false
        });
    }

    [Fact]
    public async Task Get_Index_unauthenticated_redirects_to_Login()
    {
        var response = await _client.GetAsync("/");

        response.StatusCode.Should().Be(System.Net.HttpStatusCode.Redirect);
        response.Headers.Location?.ToString().Should().Contain("/Login");
    }

    public void Dispose()
    {
        _client.Dispose();
        _factory.Dispose();
    }
}
```

**Acceptance:** New test `Get_Index_unauthenticated_redirects_to_Login` passes.
